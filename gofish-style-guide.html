✅ Real people in professional settings

AI Context Prompt
Copy this prompt at the start of any AI session to apply this style guide automatically
# Go F!sh — AI Design Context

Brand: Go F!sh — AI-enabled recruitment platform for Australian energy & renewables sector
Tagline: "Connect Globally, Hire Locally"
Audience: HR managers, energy company operators, recruitment agencies, job seekers

# Fonts
Display/Headings: Syne (weight 700–800)
Body/UI: DM Sans (weight 400–600)

# Colors
primary: #0D1B2A /* navy — authority, backgrounds */
accent: #22A078 /* green — CTAs, checkmarks, highlights */
accent-light: #2DC48F /* green light — glows, badges */
neutral: #718096 /* supporting text */
bg: #F7FAFC /* page background */

# Tone of voice
Voice: Direct, practical, optimistic — not corporate jargon
CTAs: Action-oriented — "Request Demo", "Get Started", "Join Waitlist"
Avoid: Passive voice, buzzword overload, generic SaaS phrases like "seamlessly"

# UI Patterns
Buttons: Navy (primary authority) / Green (conversion action) — border-radius 8px
Cards: White, 1px #E2E8F0 border, 20px radius, subtle shadow
Icons: Duotone SVG, navy + green only, Phosphor Icons library
Sections: Alternate white / #F7FAFC / navy backgrounds for rhythm
Spacing: 4px base unit — use 8/16/24/32/48/64px multiples only
10
UI Improvement Recommendations
Issues identified from the live site — prioritised by impact
🚨
Critical — Content
Placeholder testimonials left live
Every testimonial on the site uses "veritatis quasi architecto…" Lorem Ipsum placeholder text with the same dummy name "Charlotte Johnson, Head of eCommerce" repeated 10 times. This directly destroys trust with potential customers and looks like an unfinished template.
FixCollect 3–5 real testimonials from beta users. Format: 1–2 sentences on a specific outcome, full name, real job title, and company. Add verified logos.
🚨
Critical — Content
Leftover template section: "Prisla solutions"
The section titled "Elevate your business with our Prisla solutions" contains placeholder copy from the Webflow template it was built on. "Prisla" is not a Go F!sh product and the copy ("innovative idea Many start-ups…") is generic filler.
FixRemove or replace this section entirely. It could become a platform walkthrough, a case study block, or a demo video section.
⚠️
High — UX
No clear pricing or commitment signal
Visitors can't gauge cost or commitment level. "Request Demo" as the sole conversion path creates friction for users who want to self-serve. Many B2B SaaS companies lose conversions by only offering demo gates on beta platforms.
FixAdd a lightweight "How pricing works" section or FAQ item. Even "Free to join the waitlist — pricing announced at launch" reduces anxiety and improves conversion.
⚠️
High — Trust
Partner logos are low-contrast and undersized
The "Working with industry leading partners" section shows Acceler8, Personr, and Aussieos logos but they are small and presented without any description. Visitors don't know what these relationships mean.
FixIncrease logo sizes to minimum 120px width. Add a single-line caption per partner: e.g. "Acceler8 — Growth & Acceleration Partner". Use a light grey tinted background strip to ground the section.
⚠️
High — Visual Hierarchy
Inconsistent heading hierarchy in Platform section
The "Visa Matching" feature card uses an H2 (##) while all other feature cards use H3 (###). This creates an unintentional visual weight difference and breaks the scanning rhythm of the section.
FixNormalise all feature cards in the Platform section to H3 / the same visual weight. Apply consistent icon size (44px) and title font-size (18px) across all platform tiles.
✨
Enhancement — Modern UI
Add scroll-triggered section reveals
Currently the page renders statically. Modern SaaS sites use subtle entrance animations — cards fading up, stats counting in — to guide attention and signal polish. This is especially important for the "Why Now" stats section where the numbers are the key value proposition.
FixAdd IntersectionObserver-based reveal: opacity 0 → 1, translateY 16px → 0, 400ms ease. Stagger feature cards by 80ms. Animate stat numbers counting up on scroll entry.
✨
Enhancement — Navigation
Sticky nav should shift to light background on scroll
The hero section has a dark background but the nav sits over a light section as you scroll. A transparent-to-white transition on scroll would look significantly more polished and improve readability throughout.
FixUse a scroll event listener to toggle a .scrolled class on the nav. On .scrolled: background white, add box-shadow, switch text to navy. Transition: background 200ms ease.
✨
Enhancement — Mobile
Hamburger menu lacks visible indicator of state
On mobile, the menu toggle has no animated open/close state indicator, making it harder for users to know if the menu is open or closed. Standard expectation is a smooth X ↔ ☰ transition.
FixAnimate the hamburger icon: three lines morph to an X using CSS transform (rotate + opacity). Add a backdrop/overlay behind the mobile nav drawer.
✨
Enhancement — Content
CTA copy is too generic across multiple sections
Multiple sections use "Find out more" and "Get started now" with no specificity. These could be contextualised to the section to improve conversion — e.g. in the Visa Matching section, "Explore Visa Matching" is far more compelling than a generic CTA.
FixAudit every CTA button. Replace generic labels with action-specific alternatives: "See How Visa Matching Works", "Browse the Platform", "Talk to a Recruitment Specialist".
